---
name: validate-task
description: Use when closing a Kumoriya task and you need a strict validation pass covering format, analyze, tests, and residual risk.
---

Use this skill before declaring a Kumoriya task done.

Checklist:
- format run?
- analyze clean?
- relevant tests run?
- runtime or flow actually exercised?
- residual risks stated honestly?
- scope creep avoided?
- commit(s) clear?

Output format:
1. what changed
2. what was validated
3. what was not validated
4. residual risk
5. recommended next step
