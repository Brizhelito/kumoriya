## 22:30 — Player performance investigation (Linux desktop)

**Category:** fix

**Affected files:**
- `apps/kumoriya_app/lib/src/features/player/presentation/pages/player_page.dart`
- `apps/kumoriya_app/lib/src/features/player/presentation/widgets/player_debug_overlay.dart`

**Summary:**
User reported severe playback stutter on Linux desktop at 1080p despite mpv reporting stable 24 fps decode. Root cause: `media_kit_video` on Linux renders the video through a Flutter `Texture`; the texture is only composited when Flutter produces a frame. Without active animations, Flutter entered idle and composited ~5 fps, so the user saw the video at that rate regardless of mpv's decode rate. Additional contributor: three always-on `BackdropFilter(ImageFilter.blur)` layers in the controls overlay (play/pause σ=12, skip σ=8, speed badge σ=8) forced `saveLayer` + gaussian blur every frame.

Fixes applied:
1. Added a continuous vsync pump (`Ticker` with empty callback) in `_ImmersivePlayerViewState`, started/stopped via `didUpdateWidget` mirroring `widget.isPlaying`. Forces Flutter to schedule a frame every vsync while playing so the video texture composites at display refresh rate.
2. Removed all three `BackdropFilter` instances from the controls overlay, replaced with flat tinted containers (opacity bumped from 0x73/0xB3 to 0xCC/0xD9 to compensate for lost blur legibility).
3. Debug overlay (`PlayerDebugOverlay`) now tracks the real Flutter compositor FPS via `SchedulerBinding.addTimingsCallback` over a rolling 2 s window, with `build` vs `raster` split and `slow>16ms` / `slow>33ms` counters. This distinguishes render lag from decode lag and is reused for future profiling.

Result on Intel HD Graphics 4600 (2013 iGPU, X11, Mesa 25): UI FPS went from ~5 to ~30 stable, `build avg = 1.9 ms`, `raster avg = 13.4 ms`, zero drops. Reached the hardware ceiling for this iGPU — raster cost of compositing a 1080p texture in Flutter on HD 4600 is ~14 ms, near the 16.6 ms vsync budget, which caps pacing at 30 fps.

**Discarded attempts (reverted):**
- `VideoControllerConfiguration(height: 720)` cap + dart-define — did not take effect as expected; overlay kept reporting 1080p texture and raster did not drop.
- `gpu-hwdec-interop=auto` / `hwdec-codecs=all` via `setProperty` post-init — caused render drops and pipeline instability (fps dropped to 11). mpv ignored the interop hint anyway and kept `vaapi-copy`. Setting properties after media_kit's render API initialized re-init the decoder mid-flight.

**Residual risk / known limitation:**
- 60 fps on Linux desktop is unreachable from inside the Flutter tree on this iGPU. YouTube/VLC/mpv achieve 4K@60 via native X11/Wayland hardware overlay, bypassing the app's render pipeline. A native X11 subsurface plugin is the only path to parity; captured as a possible future epic, not part of the player slice.
- Android/iOS are expected to hit 60 fps since their texture pipelines (SurfaceTexture + HW composer) have distinct and cheaper semantics. Validation pending.
