## 16:15 — Defensive gzip handling for downloads

**Category:** fix
**Affected files:**
- `apps/kumoriya_app/lib/src/features/downloads/application/download_manager_service.dart`

**Summary:**
Some servers send `Content-Encoding: gzip` despite our `Accept-Encoding: identity` header. With `autoUncompress = false`, raw gzip bytes would corrupt video files. Added detection of gzip response encoding and on-the-fly decompression via `gzip.decoder`. When gzip is detected, `knownTotalBytes` is reset to 0 (indeterminate) since Content-Length refers to compressed size.

**Linked phase:** Phase 1.4
