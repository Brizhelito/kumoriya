## 05:45 — Fix Android SIGABRT crash on mic permission grant (null sdpMid in JNI)

**Category:** fix
**Affected files:**
- `apps/kumoriya_app/lib/src/features/watch_party/infrastructure/webrtc_peer_manager.dart`

**Summary:**
Fixed a fatal `SIGABRT` crash (`JNI DETECTED ERROR: input is not valid Modified UTF-8`) that occurred immediately after granting microphone permission on Android. Root cause: `RTCIceCandidate` was constructed with a null `sdpMid`, which `flutter_webrtc`'s native Android plugin passed to JNI `NewStringUTF`, reading garbage memory. The fix defaults `sdpMid` to `''`, coerces `sdpMLineIndex` through `num` (guarding against JSON `0.0` vs `0` type mismatch), filters null/empty candidate strings on both send and receive paths, and adds diagnostic logging to `_handleOffer`/`_handleAnswer` for malformed SDP signals.

**Linked phase:** Task 1.1 (WebRTC Voice Chat implementation)

## 02:15 — Fix persistent Android SIGABRT: skip empty-sdpMid candidates + signaling thread race guard

**Category:** fix
**Affected files:**
- `apps/kumoriya_app/lib/src/features/watch_party/infrastructure/webrtc_peer_manager.dart`
- `apps/kumoriya_app/lib/src/features/watch_party/application/providers/voice_providers.dart`

**Summary:**
The previous null-sdpMid fix (defaulting to `''`) was insufficient — passing an empty `sdpMid` to native `IceCandidate()` still triggered `NewStringUTF` on garbage memory when the native WebRTC stack failed to resolve the empty media section reference. This fix **skips** ICE candidates entirely when `sdpMid` is null or empty (they are unusable without a valid `m=` section). Additionally, a 100ms delay was inserted between `setRemoteDescription` and flushing buffered candidates to prevent a signaling-thread race condition, and peer connections are now staggered by 200ms during initial setup to reduce native thread contention.

**Linked phase:** Task 1.1 (WebRTC Voice Chat implementation)
