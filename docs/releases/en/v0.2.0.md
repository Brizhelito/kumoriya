# Kumoriya v0.2.0 — Alpha Release Notes

**Date:** 2026-04-18

## Summary

Minor release focused on Watch Party, backend-powered discovery/alerts, and a more reliable API-driven release distribution flow.

## Added

- **Dedicated Watch Party surfaces** — party anime and episode pages now prioritize room context, members, readiness state, and the collaborative playback flow.
- **Backend AniList home cache** — trending, seasonal discovery, and airing calendar are now served through `api.kumoriya.online` with conservative caching and transparent fallback.
- **FCM push notifications for airing episodes** — per-anime topic subscriptions now follow the library toggle and are reconciled after login and on every boot.
- **API-backed release feed** — both the app and website now read release metadata from the Neon-backed API.

## Changed

- **Watch Party navigation** — detail, episode list, and player flows now preserve room context and route users back to the lobby on exit.
- **Profile page hierarchy** — Watch Party is now much more prominent, logout is easier to find, and delete-account/UUID elements are visually de-emphasized.
- **Legacy new-episodes worker** — local notifications were retired in favor of FCM push and the worker now focuses on auto-download traffic only.
- **Website release source** — download links and changelog entries now come from the API instead of hand-maintained JSON as the primary source.

## Fixed

- **Release publish staleness** — publishing a new version no longer depends on a one-shot cached manifest; the API refreshes its release snapshot on publish.
- **Notification channel drift** — Android and backend now share the same `kumoriya_new_episodes` channel.
- **Accidental account deletion risk** — destructive actions in Profile are now clearly secondary to the main session actions.
