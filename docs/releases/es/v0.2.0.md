# Kumoriya v0.2.0 — Notas de Versión Alpha

**Fecha:** 2026-04-18

## Resumen

Versión menor enfocada en Watch Party, descubrimiento/alertas impulsadas por backend y una distribución de releases más sólida a través de la API.

## Agregado

- **Superficies dedicadas de Watch Party** — las páginas de anime y episodios de la party ahora priorizan el contexto de sala, miembros, estado de preparación y el flujo colaborativo de reproducción.
- **Cache backend para el home de AniList** — tendencias, temporada y calendario de emisión ahora se sirven desde `api.kumoriya.online` con cache conservadora y fallback transparente.
- **Push notifications FCM para episodios al aire** — las suscripciones por anime ahora siguen el toggle de biblioteca y se reconcilian después del login y en cada arranque.
- **Feed de releases por API** — la app y la website ahora leen metadata de releases desde la API respaldada por Neon.

## Cambios

- **Navegación de Watch Party** — el flujo de detalle, lista de episodios y reproductor conserva el contexto de sala y vuelve al lobby al salir.
- **Jerarquía de la pantalla de perfil** — Watch Party ahora es mucho más visible, logout ganó protagonismo y borrar cuenta/UUID quedaron más discretos.
- **Worker legacy de nuevos episodios** — las notificaciones locales fueron retiradas a favor de FCM push y el worker quedó centrado en auto-descargas.
- **Website conectada al feed de releases** — enlaces de descarga y changelog ahora se generan desde la API en lugar de depender de JSON mantenido a mano.

## Corregido

- **Staleness del publish de releases** — publicar una versión nueva ya no depende de un manifiesto cacheado una sola vez; la API refresca su snapshot al publicar.
- **Deriva del canal de notificaciones** — Android y backend ahora usan el mismo canal `kumoriya_new_episodes`.
- **Riesgo de borrado accidental de cuenta** — las acciones destructivas en Perfil quedaron claramente relegadas frente a las acciones principales.

## Notas de Migracion

- Ninguna.
